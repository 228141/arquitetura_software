Arquitetura Descentralizada

Cliente-servidor possuem duas distribuições:
  • Ditribuição Vertival
    • Componentes logicamente diferentes em máquinas diferentes
    • Cada máquina vai executar um conjunto especifico e pré determinado de funções
    • As funções de cada um são bem definidas
  • Distribuição Horizontal
    • Cliente ou servidor pode ser fisicamente subidividido em partes logicamente equivalemte
    • Pode possuir porção de dados
    • Balanceamento do carga
    • Papéis não muito bem definidos
    • Ex: Peer-to-Peer que envia um "pedaço" de um arquivo

Arquitetura Descentralizada - Peer-to-Peer

  • Processos são todos iguais e flat
  • Cada processo age ao mesmo tempo como cliente e servidor(servente)
  • O cliente é quem inicia a requisição-resposta

Rede de sobreposição: rede onde os nós são formados pelos processos e os enlaces denotam os canais de comunicação.

Arquiteturas "estruturadas" e "não-estruturais"

Arquitetura Estruturadas

  • Rede de sobreposição é construída usuando um mecanismo determinístico e estruturado
  • Mecanismo usando comumente: DHT (Distributed Has Table).
  • Dados e nós recebem uma chave aleatória (128^160 bits).

    • Desafio: dada uma chave de um dado, papear para o identificador de um nó.

  • Neste caso a requisição é roteada entre os nós até alcançar o nó com o dado solicitado.
