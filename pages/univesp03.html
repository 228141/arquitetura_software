Estilo Arquitetônico

Componentes = unidade modular com interfaces REQUIRIDAS e FORNECIDAS bem definidas que é substituivel dentro do seu ambiente. Pode ser criado conexões flexivel atraves de componentes dos modulos. Conexoes = o modo como os componentes estão ligados Dados Intercambio = forma de troca dos dados entre os componentes (repositório compartilhado?) Forma de configuração = maneira como os componentes são configurados (em tempo de execução?)

Tipos de estilo Arquitetônico

  • Arquitetura de camadas
  • Arquitetura baseada em objetos
  • Arquitetura centrada em dados
  • Arquitetura baseada em eventos

arquitetura de sistemas

Arquitetura Centralizada = Modelo Cliente Servidor
    Processos são divididos em dois grupos Servidor: Processo que imlplementa um serviço especifico Cliente: processo que requisita um serviçço ao servidor forma de interação: requisição-resposta
Camadas de Aplicação = Considerando muitas aplicações e a sua escalabilidade:
    Nível de interface do usuário Nivel de processamento Nivel de dados
Arquitetura Centralizada com arquiteturas multidivididas Gerenciamento de sistemas:
    Clientes gordos (Fat clients) Clientes magros (thin clients)
Distinção Clara: Arquitetura de duas divisões (cliente e servidor) Arquitetura de três divisões:
    Cliente Servidor Servidor que pode agir como cliente
