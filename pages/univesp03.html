Estilo Arquitetônico

Componentes = unidade modular com interfaces REQUIRIDAS e FORNECIDAS bem definidas que é substituível dentro do seu ambiente. Pode ser criado conexões flexíveis através de componentes dos módulos.

  • Conexões = o modo como os componentes estão ligados
  • Dados Intercâmbio = forma de troca dos dados entre os componentes (repositório compartilhado?)
  • Forma de configuração = maneira como os componentes são configurados (em tempo de execução?)

Tipos de Estilo Arquitetônico

  • Arquitetura de camadas
  • Arquitetura baseada em objetos
  • Arquitetura centrada em dados
  • Arquitetura baseada em eventos

Arquitetura de Sistemas

Arquitetura Centralizada = Modelo Cliente/Servidor: Netflix, sites de notícias, on-line bank, etc.

  • Processos são divididos em dois grupos: Servidor e Cliente;
  • Servidor: Processo que implementa um serviço específico;
  • Cliente: processo que requisita um serviço ao servidor;
  • Forma de interação: requisição-resposta.

Camadas de Aplicação = Considerando muitas aplicações e a sua escalabilidade:

  • Nível de interface do usuário
  • Nível de processamento
  • Nível de dados

Arquitetura Centralizada com arquiteturas multidivididas

Gerenciamento de sistemas:

  • Clientes gordos (Fat clients)
  • Clientes magros (Thin clients)

Distinção Clara: Arquitetura de duas divisões (cliente e servidor)

Arquitetura de três divisões:

  1. Cliente
  2. Servidor
  3. Servidor que pode agir como cliente
