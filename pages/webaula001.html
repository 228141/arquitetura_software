Introdução

Antes de começar um projeto de software, é necessário definir qual será a ESTRUTURA e a ORGANIZAÇÃO do PROJETO, este é o primeiro estágio de qualquer projeto sendo chamado ARQUITETURA DE SOFTWARE.

Vamos pensar na arquitetura de software como a planta de uma casa. Quando alguém quer construir uma casa, não começa simplesmente colocando tijolos aleatoriamente. Primeiro, faz-se uma planta, que é um desenho que mostra como a casa vai ser, onde estarão os quartos, banheiros, a cozinha, etc. A arquitetura de software é como essa planta, mas em vez de ser para uma casa, é para um programa de computador. Ela mostra como o software será organizado, como as diferentes partes dele vão trabalhar juntas e quais materiais (ou tecnologias) serão usados para construí-lo. Por exemplo, imagine que você quer construir um jogo. A arquitetura do software vai dizer onde o jogo vai guardar os pontos dos jogadores, como os personagens vão se mover na tela e como o jogo vai entender quando alguém ganha ou perde. Assim como a planta de uma casa ajuda os pedreiros a saber onde colocar cada coisa, a arquitetura de software ajuda os programadores a construir o programa de maneira organizada e eficiente. Começamos com essa planta, ou seja, a arquitetura, para evitar problemas no futuro e para garantir que tudo funcione bem junto, assim como uma casa bem planejada é confortável e segura para viver.
